# LaTeXUkrainianOnFedoraLinux
Версія Fedora, на якій це перевірялось: 43
Редактор LaTeX: TeXstudio

Перш за все, потрібно поставити саму TeXstudio на ваш ПК:
```bash
sudo dnf install texstudio
```
Потім, при відкритті .tex-файла, якщо він містить кириличні символи, потрібно змінити кодування через Edit -> Setup Encoding, обрати "windows-1251 / ibm-5347_P100-1998 / ibm-5347 / cp1251 / ANSI1251" і кнопку View With.

Відкриється файл уже в кодуванні, windows-1251, кириличні символи відображатимуться нормально. А от компіляція, можливо і не пройде так легко! В логах будуть "скарги" на те, що не вийшло знайти файли для кодування (наприклад cp1251.def), або що babel вирішив, що в слові ukrainian є помилка. Для цього потрібно поставити наступні пекеджі, які не ставляться автоматично разом із TeXstudio:

```bash
# Для babel
sudo dnf install texlive-babel-ukrainian
# Для помилки Font T2A/cmr/m/n/10=larm1000 at 10.0pt not loadable: Metric (TFM) file not found.
sudo dnf install texlive-lh
# Аби не сварилось, що не знайшло контрол просто компілюючи текст українською
sudo dnf install texlive-cyrillic
# цей пекедж просто корисний для переносів слів між рядками, без нього скомпілюється, але без переносів слів
sudo dnf install texlive-ukrhyph

```

Якщо вам, упаси Господь, потрібні ще пекеджі для російської (для компіляції наданого мені шаблону, на жаль, потрібні були), то ставите texlive-babel-russian і texlive-ruhyphen.

Взагалі, мотивація для цього гайду була в тому, що на Ubuntu (і загалом debian-derived дистрибутивах) і Fedora однакові болячки з цим, але різні назви пекеджів, тож накидав це нашвидкоруч, аби, можливо, комусь ще це було корисним.
